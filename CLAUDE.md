# blesk — Project Brief for Claude Code

## Обзор проекта

**blesk** — приватный десктопный мессенджер с открытым исходным кодом. Объединяет приватность Signal, социальность Discord, каналы Telegram и игровую экономику. Часть экосистемы бренда GOTBLESK.

### Брендинг
- **Название:** blesk (всегда маленькими буквами)
- **Слоган:** "Твой блеск. Твои правила."
- **blesk+** (подписка) — слоган: "Блести ярче"
- **Логотип:** "bl" в стиле Liquid Glass (маленькие буквы)
- **Написание в интерфейсе:** blesk (не BLESK, не Blesk, не BLESKCHAT)
- **Написание в коде:** blesk (имена файлов, переменные, пакеты)
- **Домен:** blesk.fun
- **GitHub (план):** github.com/blesk-app

**Основатель:** Егор Макаров (GOTBLESK), 20 лет, дизайнер с 7-летним опытом (Photoshop, After Effects, Premiere Pro), не программист — использует AI-инструменты для разработки.

**Платформа:** Windows (Desktop), распространение через GitHub.

## Технический стек

- **Клиент:** Electron + React (MVP), миграция на Tauri планируется в v2.0
- **Бэкенд:** Node.js + Express
- **Реалтайм:** WebSocket (Socket.io)
- **База данных:** PostgreSQL
- **Голос/видео:** WebRTC + mediasoup (SFU)
- **Шифрование:** tweetnacl / libsodium (E2E для личных чатов)
- **Авторизация:** JWT (15 мин) + refresh tokens + bcrypt для паролей
- **Сборка:** electron-builder (NSIS установщик + portable)
- **Платежи:** ЮKassa (планируется)
- **Хостинг:** VPS за пределами РФ (Hetzner), Cloudflare для DDoS-защиты

## Дизайн

### Стиль: Liquid Glass (Apple iOS 26 inspired)

- Многослойное стекло: backdrop-filter blur + полупрозрачный фон + specular highlights + edge glow + outer shadow
- 4 уровня глубины стекла (depth 0-3) для визуальной иерархии
- Анимированный фон с цветными orbs, реагирующими на движение мыши
- Кислотно-зелёный акцент (#c8ff00) в тёмной теме — фирменный цвет GOTBLESK
- Шрифты: системные (-apple-system, SF Pro Display, Helvetica Neue)
- Тёмная и светлая темы + режим экономии (без blur, без анимаций)
- **Навигация:** горизонтальная glass-полка сверху с pill-табами (НЕ Discord-сайдбары)
- Боковая панель (люди/настройки) выезжает по клику, не постоянная

## Фичи пост-релиза (подробно)

### Бот-платформа (blesk api)
Открытый API для создания ботов — как в Discord и Telegram.
Любой разработчик может написать бота и добавить в группу/канал.

**Что бот может делать:**
- Читать сообщения в чате (с разрешения админа)
- Отправлять сообщения, картинки, embed-карточки
- Реагировать на команды (/ban, /play, /roll, кастомные)
- Управлять ролями (выдавать/снимать по условиям)
- Модерировать (автомут за мат, антиспам, приветствие новых)
- Интегрироваться с внешними API (погода, музыка, игры)
- Взаимодействовать с экономикой (начислять монеты за активность через API)

**Типы ботов которые появятся:**
- Модерация — автобан, антиспам, логи (как MEE6 в Discord)
- Музыкальные — воспроизведение треков в голосовой
- Игровые — /roll, /coinflip, мини-розыгрыши
- Утилиты — напоминания, таймеры, погода, курсы валют
- Интеграции — GitHub, Twitch, YouTube уведомления

**Как создать бота:**
1. Админ заходит в настройки → "Разработчикам" → "Создать бота"
2. Получает токен (API key) для авторизации бота
3. Бот подключается к серверу через WebSocket (как обычный клиент, но с ограничениями)
4. Документация API на сайте blesk (Swagger/OpenAPI)
5. SDK на JavaScript (npm-пакет `@blesk/bot-sdk`)

**Безопасность ботов (КРИТИЧНО):**

Принцип: бот — это внешняя программа на компьютере создателя, которая общается
с blesk ТОЛЬКО через API. Бот НЕ выполняет код на нашем сервере.
Бот не имеет доступа к файловой системе, базе данных, другим ботам.

Уровень 1 — Права (админ выбирает при добавлении):
- Каждое право отдельный чекбокс: читать, писать, модерировать, управлять ролями
- Нет галочки "модерировать" = бот физически не может банить

Уровень 2 — Лимиты:
- 30 сообщений/мин, 5 embed-карточек/мин, 10 действий модерации/мин
- Превышение = автоблокировка на 10 мин + уведомление админу

Уровень 3 — Мгновенное отключение:
- Кнопка "Отключить бота" = токен аннулирован, доступ потерян за секунды

Уровень 4 — Каталог с модерацией:
- Bot Store — боты проверяются перед публикацией
- Непроверенные боты: жёлтый значок "Не верифицирован" + предупреждение

Уровень 5 — Жалобы:
- Кнопка "Пожаловаться на бота" у каждого бота
- N жалоб = автоотключение во всех группах до проверки

Уровень 6 — Логирование:
- Все действия бота записываются, можно откатить разом

Защита от конкретных атак:
- Спам → rate limiting обрежет + уведомление админу
- Кража токенов → API не выдаёт чужие токены, хранятся в httpOnly cookie
- Чтение личных сообщений → E2E, API не предоставляет доступ
- Фишинговые ссылки → фильтр URL + предупреждение "ссылка от непроверенного бота"
- DDoS сервера → rate limiting + Cloudflare + блокировка токена
- Кража данных → API не выдаёт ботам телефоны, email, IP пользователей
- Вирус через файл → запрет .exe/.bat/.sh + предупреждение при скачивании

Предупреждения в интерфейсе:
- При добавлении бота: "Бот получит доступ к сообщениям в этом канале"
- Непроверенный бот: жёлтый значок "Не верифицирован"
- Скачивание файла от бота: "Файл от бота. Скачивайте если доверяете"

**Каталог ботов (Bot Store):**
- Раздел в приложении — как обзор каналов, только для ботов
- Категории: модерация, музыка, игры, утилиты, интеграции
- Рейтинг, отзывы, количество использований
- Популярные боты привлекают пользователей на платформу
- Создатель бота получает тег "Разработчик ботов"

**Монетизация:**
- Создание бота — бесплатно (привлекаем разработчиков)
- Добавление бота в группу — бесплатно (до 3 ботов)
- Больше 3 ботов — 200 монет за слот
- Премиум-боты от создателей — платные (создатель ставит цену, blesk берёт 10-15% комиссии)
- Продвижение в каталоге — 500-1500 монет в неделю
- API-лимиты: бесплатно 1000 запросов/час, blesk+ бот-тариф: 10000 запросов/час (для крупных ботов)

### Пространства (Spaces)
Уровень выше каналов — объединение нескольких каналов и голосовых в одно сообщество.
Пространство "GOTBLESK" → внутри #новости, #дизайн, #музыка, голосовая "студия".
У каждого Пространства свой конструктор, своя кастомизация, свои роли.
Пользователь может состоять в нескольких Пространствах и переключаться.

**Лимиты для групп и пространств (бесплатно):**
- 8 текстовых чатов
- 8 голосовых комнат
- Итого 16 комнат — достаточно для большинства сообществ
- Больше — за монеты: +5 текстовых = 500 монет, +5 голосовых = 500 монет
- blesk+ / "Канал+": безлимит комнат

Монетизация: создание Пространства бесплатно (лимит 3), больше — за монеты.
Кастомизация Пространства — через конструктор (пакеты "Стартер"/"Про"/"Ультра").

### Моменты (Stories)
Пользователь публикует фото, текст или короткое аудио — видно друзьям 24 часа.
Лента моментов — горизонтальная полоска с аватарами вверху (как Instagram).
Реакции на моменты — эмодзи, быстрый ответ.
Монетизация: эффекты для моментов (рамки, фильтры, анимации) за монеты.
Премиум-моменты: видео до 30 сек, кастомные фоны — blesk+.
Моменты каналов: админ канала публикует момент → все подписчики видят.

### Радио (фоновый музыкальный режим)
Создатели каналов загружают треки, формируют плейлисты.
Подписчики включают "Радио" — музыка играет в фоне.
Чарт "Топ-10 треков недели в blesk".
Для андеграунд-музыкантов — платформа для дистрибуции.
Связка с Listen Together — совместное прослушивание в голосовой.
Монетизация: загрузка треков бесплатна, продвижение в чарте — за монеты.
Примечание: учитывать авторские права — только оригинальный контент авторов.

### Soundboard (звуковые команды)
В голосовой комнате — панель быстрых звуков (аплодисменты, барабанная дробь, смех, сирена).
Кнопка или команда — звук играет для всех в комнате.
Базовый набор (10 звуков) — бесплатно.
Дополнительные паки — за монеты (50-200 монет за пак).
Авторские паки от основателя (используя аудио-опыт) — премиум.
Пользователи могут загружать свои звуки (модерация обязательна).

### Доска настроения канала (Mood Board)
Виджет для конструктора каналов.
Анализирует эмодзи-реакции и настроение сообщений за последний час.
Визуально: анимированный индикатор (огонь = активно, облако = спокойно, молния = жарко).
Меняется в реалтайме — живой элемент канала.
Бесплатный виджет — привлекает к конструктору.

### Интеграция с играми
Electron определяет запущенные процессы (через process list).
Автоматический статус: "Играет в CS2", "Играет в Valorant".
LFG (Looking For Group): "3 твоих друга играют в CS2 прямо сейчас — присоединиться в голосовую?"
Настраиваемый список игр (пользователь выбирает какие показывать).
Отключаемо в настройках приватности.

### Тихий час + утренний дайджест
"Тихий час" — не просто отключение уведомлений, а умный режим:
- Копит все сообщения и события
- Утром показывает красивый экран-дайджест:
  "Пока вы отдыхали: 47 сообщений в #основном, Катя написала вам,
   в голосовой было 8 человек, новый пост в #музыка"
- Автоматическое расписание (например 23:00—08:00)
- Или ручное включение

## Экономика blesk coins — правила

### Базовый курс (ФИКСИРОВАННЫЙ, НЕ МЕНЯЕТСЯ НИКОГДА)
- 1 рубль = 10 монет
- Пакеты: 100₽=1000, 250₽=2800(+12%), 500₽=6000(+20%), 1000₽=13000(+30%)

### Две системы торговли

**Магазин blesk (фиксированные цены):**
- Цены устанавливает основатель, они НЕ меняются
- Пользователь покупает → монеты СГОРАЮТ (уходят из системы)
- Некоторые предметы лимитированные (по количеству или по времени)
- Когда лимит закончился — предмет пропадает из магазина навсегда
- Ценовые категории (не менять):
  - Базовые: 200–500 монет (ауры, стикерпаки, простые эффекты)
  - Редкие: 1000–2000 монет (редкие ауры, анимированные эффекты)
  - Легендарные: 3000–5000 монет (лимитированные, сезонные)
  - Новые предметы всегда в этих диапазонах — НЕ поднимать цены

**Маркетплейс (свободные цены между пользователями):**
- Продавец сам устанавливает цену
- Покупатель сам решает — брать или нет
- Лимитированная аура из магазина за 3000 → на маркетплейсе может быть 10000+
- Аукцион ников — цену определяют ставки покупателей
- Комиссия blesk: 10–15% с каждой сделки (монеты СГОРАЮТ)
- Графики цен: средняя цена предмета за месяц (как на бирже)

### Смена никнейма
- 1 бесплатная смена в месяц (каждому пользователю)
- Дополнительные смены: 1000 монет за каждую
- blesk+ пользователи: 2 бесплатные смены в месяц

### Анти-инфляция (5 механизмов)

**1. Стоки (sinks) — где монеты сгорают:**
- Покупки в магазине, бусты каналов, продвижение, комиссии маркетплейса
- Смена ника, конструктор каналов, дополнительные слоты ботов
- ПРАВИЛО: каждая новая фича должна включать трату монет

**2. Лимит бесплатных монет:**
- Ежедневный вход: 10 монет (макс 300/мес)
- Рефералы: 200 монет (макс 20 рефералов = 4000 монет, потом стоп)
- Достижения: разовые, не повторяются
- Квесты: фиксированная награда, не растёт

**3. Подписки — постоянный отток:**
- Буст канала: 500 монет/мес (нужно обновлять)
- "Канал+": 149 руб/мес
- Продвижение: оплата за неделю

**4. Лимитированные дропы:**
- Регулярные выпуски новых предметов = массовая трата монет
- Сезонные (Новый Год, Хэллоуин) = все бросаются покупать

**5. Мониторинг в админ-панели:**
- Дашборд: монет входит/день vs монет выходит/день
- Если вход > выход на 30%+ — выпустить новый предмет или уменьшить бесплатную раздачу
- Целевой баланс: вход ≈ выход ± 15%

### ЗАПРЕЩЕНО (золотые правила экономики)
- НЕ привязывать монеты к крипте или реальной валюте с плавающим курсом
- НЕ разрешать вывод монет в рубли (юридические проблемы + спекуляция)
- НЕ начислять основателю миллионы монет для продажи
- НЕ поднимать цены в магазине на существующие предметы
- НЕ менять базовый курс 1₽ = 10 монет
- НЕ выпускать повторно лимитированные предметы (убивает ценность коллекций)

### Пирамида редкости предметов

Цены в магазине НЕ растут — появляются новые более дорогие категории.

| Редкость | Цвет | Цена (монеты) | Цена (руб) | Доступность |
|----------|------|---------------|------------|-------------|
| Обычные | Серый | 200–500 | 20–50 | Всегда в магазине |
| Необычные | Зелёный | 500–1000 | 50–100 | Ротация, меняются каждую неделю |
| Редкие | Синий | 1000–3000 | 100–300 | Сезонные, ограниченное время |
| Эпические | Фиолетовый | 3000–10000 | 300–1000 | Лимит по количеству (500 шт) |
| Легендарные | Золотой | 10000–30000 | 1000–3000 | Раз в 2–3 мес, лимит 100 шт |
| Мифические | Красный | 30000–50000 | 3000–5000 | Раз в полгода, лимит 50 шт, с номером (#N из 50) |
| Божественные | — | Не продаются | — | Выдаются основателем лично за исключительные заслуги. ~5 шт за всю историю |

Номера предметов: лимитированные предметы имеют порядковый номер (#1 из 50).
Ранние номера ценнее на маркетплейсе — мотивация покупать быстрее.

### Статус аккаунта (публичная метрика)

В профиле каждого пользователя отображается:
- Количество предметов в коллекции
- Общая редкость (средняя по всем предметам)
- Оценочная стоимость коллекции (по текущим ценам маркетплейса)
- Рейтинг уникальности (по тегам, 0–100)
- **Статус аккаунта** — общая метрика:

| Статус | Цвет | Как получить |
|--------|------|-------------|
| Бронзовый | Коричневый | Регистрация + базовый профиль |
| Серебряный | Серый | 5+ предметов, 10+ тегов |
| Золотой | Золотой | 15+ предметов, коллекция 10000+ монет |
| Платиновый | Белый | 30+ предметов, коллекция 50000+ монет, 6+ мес аккаунт |
| Бриллиантовый | Голубой | 50+ предметов, коллекция 200000+ монет, 1+ год, 50+ тегов |

Статус зависит от: коллекции + тегов + активности + возраста аккаунта.
Нельзя просто купить — нужны и деньги, и время, и активность.
Статус виден всем рядом с ником (иконка/рамка).

### Психология трат (почему люди платят больше)

**FOMO (страх упустить):**
- "Осталось 12 из 50 Мифических аур" — уведомление в магазине
- Лимитированные предметы НЕ возвращаются в магазин никогда
- Сезонные предметы доступны только в своём сезоне

**Социальное давление:**
- Друг купил ауру → ты видишь её в каждом чате → хочешь такую же
- Лидерборд коллекций — топ-50 самых дорогих аккаунтов

**Коллекционирование:**
- "У меня 47 из 60 тегов" → хочу все
- "Все сезонные ауры с момента запуска" → не пропущу следующую

**Ощущение инвестиции:**
- Потратил 10000₽ → аккаунт "стоит" 300000 монет на маркетплейсе
- Предметы растут в цене → человек чувствует что инвестировал, а не потратил

**Номера предметов:**
- "#3 из 50" круче чем "#48 из 50"
- Ранние номера дороже на маркетплейсе
- Мотивация покупать быстрее при дропе

1. **Безопасность:** Параметризованные запросы (Prisma), никогда не доверять клиенту, rate limiting на все эндпоинты, валидация всех входных данных

4. **UX:** Анимации через CSS transitions (не JS), мгновенный отклик на действия, оптимистичные обновления UI
5. **Код:** ESLint + Prettier, осмысленные git-коммиты, README на русском и английском

## Конструктор каналов (Channel Builder) — Beta

### Концепция
Безопасный конструктор для админов каналов. Два режима:
1. **Визуальный (основной)** — drag-and-drop, мышкой перетаскивай блоки
2. **Терминальный (продвинутый)** — текстовые команды из закрытого списка

Визуальный режим — по умолчанию. Терминальный — включается отдельно для тех кто хочет.

### Что админ может настроить
- **Лейаут**: чат + сайдбар, полноэкранный чат, чат + виджеты справа
- **Виджеты** (готовые блоки из каталога):
  - Закреплённые сообщения, правила канала, расписание ивентов
  - Статистика канала, медиа-галерея, список модераторов
  - Кастомные ссылки, голосование/опрос, приветствие новым
- **Стиль канала**: цветовая тема, фон чата, шрифт, форма аватарок, стиль сообщений
- **Навигация**: порядок секций, видимость элементов

### Визуальный режим (основной, для всех)
Интерфейс как Figma/Tilda — перетаскивание мышкой:
- Левая панель: каталог виджетов (иконка + название + описание)
- Центр: превью канала в реальном времени (WYSIWYG — что видишь то и получаешь)
- Правая панель: настройки выбранного элемента (цвет, текст, размер)
- Перетаскивание виджета из каталога → на превью → виджет появляется
- Перетаскивание виджетов внутри превью → меняют порядок и позицию
- Клик на виджет → правая панель показывает его настройки
- Кнопка "Сохранить" и "Превью" (показать как видят участники)
- Кнопка "Отменить изменения" → откат к последнему сохранению

### Терминальный режим (для продвинутых)
Включается через кнопку "Терминал (Beta)" в конструкторе.
Те же настройки, но через текстовые команды:
```
/layout sidebar-right
/widget add rules
/widget remove media-gallery  
/theme bg #1a1530
/theme accent #c8ff00
/welcome Добро пожаловать!
/preview
/history — показать историю изменений
/rollback 3 — откатить к версии 3
```
Автозаполнение + подсказки. Закрытый список команд.

### Безопасность
- Админ НЕ пишет код — перетаскивает блоки или выбирает из списка команд
- Все значения валидируются на сервере (hex-цвета, допустимые шрифты, размеры)
- Виджеты — закрытый каталог, нельзя создать произвольный
- Настройки хранятся как JSON: `{"layout": "sidebar-right", "widgets": [...], "theme": {...}}`
- Никакого исполняемого кода — только данные

### Система предупреждений (три уровня)

**Зелёный (безопасно)** — без предупреждений:
- Смена цвета акцента, добавление виджета, изменение текста приветствия
- Мелкие изменения, легко откатить

**Жёлтый (внимание)** — мягкое предупреждение:
- Смена лейаута, удаление виджетов, изменение стиля сообщений
- Модалка: "Это изменение увидят все участники. Хотите сначала превью?"
- Кнопки: "Превью" / "Применить"
- В терминале: жёлтый текст + требует `/command confirm`

**Красный (критично)** — серьёзное подтверждение:
- Сброс всех настроек, применение чужого пресета поверх текущего
- Модалка с красным акцентом: "Это перезапишет ВСЕ настройки. Необратимо."
- Поле ввода: "Введите название канала для подтверждения"
- В терминале: красный текст + `/reset-all confirm [название канала]`

**Предупреждение при входе в терминальный режим:**
- Модалка: "Режим расширенных настроек. Изменения применяются мгновенно. Используйте /preview и /history."
- Чекбокс "Больше не показывать"

### История изменений (ctrl+z для канала)
- Каждое сохранение создаёт версию в истории
- Админ может просмотреть все версии и откатить на любую
- Бесплатно: последние 5 версий
- blesk+: безлимит версий
- В визуальном режиме: кнопка "История" → список версий с датами
- В терминале: `/history` → список, `/rollback N` → откат

### Тег "Терминальщик"
- Скрытый тег-достижение
- Автоматически выдаётся после 10+ настроек через терминальный режим
- Видно в коллекции тегов пользователя
- Мотивирует пробовать терминальный режим

- **blesk.fun** — купить на Beget (всё в одной панели)
- Cloudflare (бесплатно) — DNS, DDoS-защита, SSL

### Где что живёт

**GitHub (бесплатно):**
- Исходный код проекта (репозиторий)
- CI/CD через GitHub Actions (автосборка .exe)
- Releases — .exe файлы для скачивания пользователями

**Beget VPS Европа:**
- Node.js сервер (API + WebSocket)
- PostgreSQL (база данных)
- Mediasoup (голос/видео)
- Nginx (входная точка, проксирование)
- Ubuntu 24

**Cloudflare (бесплатно):**
- DNS (blesk.fun → IP сервера)
- DDoS-защита
- SSL-сертификат (https)

**Компьютер пользователя:**
- Electron-приложение (.exe)
- Приватные ключи E2E шифрования

### Деплой (как заливать код на сервер)

**Во время разработки:**
- Всё работает локально на ПК (localhost)
- Тестирование с друзьями — через ngrok (бесплатный туннель)

**Когда MVP готов:**
- git push → GitHub Actions автоматически:
  1. Запускает тесты
  2. Собирает .exe (electron-builder)
  3. Деплоит серверную часть на Beget VPS через SSH
  4. Публикует .exe в GitHub Releases
- Один раз настроил — дальше всё автоматически

### Миграция (когда понадобится)

Архитектура проекта — Docker-контейнеры.
Миграция на любой хостинг: упаковал → поднял на новом сервере → переключил DNS → готово.
Пользователи не заметят. Время миграции: 1–2 часа.

## Важные заметки

- Основатель работает с мобильного устройства и десктопа
- Основатель — дизайнер, не кодер. Код должен быть понятным и хорошо комментированным
- Все объяснения и комментарии на русском языке
- Проект open-source (AGPL-3.0), код должен быть чистым для публичного просмотра
- При генерации кода избегать типичных AI-паттернов (избыточные комментарии, шаблонные имена)
- Версионирование: SemVer (MAJOR.MINOR.PATCH) + стадия (-alpha, -beta, -rc)
